# Python

Place Python research code here.

Suggested practices:

- Use clear script and notebook names.
- Keep reusable functions in modules rather than copying code between notebooks.
- Record important package requirements in a `requirements.txt`, `environment.yml`, or project README.
- Do not commit virtual environments such as `.venv/` or `venv/`.

Possible structure for a larger Python project:

```text
project-name/
  notebooks/
  src/
  tests/
  results/
  README.md
```
